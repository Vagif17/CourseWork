# Документация Доменного Слоя в TextMe (Domain)

В этом документе описаны основные сущности и бизнес-логика ядра приложения **TextMe**. Доменный слой спроектирован с использованием принципов чистой архитектуры (Clean Architecture) и является независимым от внешних библиотек и баз данных (за исключением ASP.NET Identity).

---

## 1. Chat (Чат)
Класс `Chat` представляет собой единицу общения. Это может быть как личная переписка (private), так и групповой чат.

| Свойство | Тип | Описание |
| :--- | :--- | :--- |
| **Id** | `int` | Уникальный идентификатор чата. |
| **Name** | `string` | Название чата (для групп) или имя собеседника. |
| **IsGroup** | `bool` | Флаг, указывающий, является ли чат групповым. |
| **Participants** | `ICollection` | Список всех участников чата. |
| **GroupAvatarUrl** | `string?` | Ссылка на аватарку группы. |
| **CreatedAt** | `DateTimeOffset` | Дата и время создания чата. |

> [!NOTE]
> **Денормализация:** Свойства `LastMessageAt` и `LastMessagePreview` являются результатом денормализации. Это сделано для того, чтобы быстро получать список чатов с предпросмотром последнего сообщения без тяжелых SQL-запросов (Join) к таблице сообщений.

---

## 2. Message (Сообщение)
Представляет собой сообщение внутри чата. Поддерживает текстовый и медиа контент.

| Свойство | Тип | Описание |
| :--- | :--- | :--- |
| **Id** | `int` | Уникальный ID сообщения. |
| **ChatId** | `int` | Ссылка на чат, к которому относится сообщение. |
| **SenderId** | `string` | ID пользователя, отправившего сообщение. |
| **Text** | `string?` | Текстовое содержимое сообщения. |
| **MediaUrl** | `string?` | Ссылка на медиа-файл (изображение, аудио, видео). |
| **MediaType** | `string?` | Тип медиа (например, "image/png", "audio/mpeg"). |
| **AudioDuration** | `int?` | Длительность аудио сообщения в секундах. |
| **Status** | `Enum` | Статус (Отправлено, Доставлено, Прочитано). |
| **ReplyToMessageId**| `int?` | Ссылка на сообщение, на которое был дан ответ (Threading). |
| **IsDeleted** | `bool` | Отметка о «удалении» сообщения (Soft Delete). |
| **IsEdited** | `bool` | Флаг, указывающий, было ли сообщение отредактировано. |

---

## 3. ChatParticipant (Участник чата)
Это связующее звено между пользователями и чатами (связь "Многие-ко-Многим").

| Свойство | Тип | Описание |
| :--- | :--- | :--- |
| **ChatId** | `int` | Ссылка на чат. |
| **UserId** | `string` | Ссылка на пользователя. |
| **IsAdmin** | `bool` | Имеет ли пользователь права администратора в этом чате. |

---


## 4. Вспомогательные сущности

### MessageStatus (Перечисление)
```csharp
public enum MessageStatus {
    Sent,      // Отправлено (на сервере)
    Delivered, // Доставлено (на устройство получателя)
    Read       // Прочитано (пользователь открыл чат)
}
```

### RefreshToken (Безопасность)
Используется для обновления JWT-токенов без необходимости повторного ввода пароля. Хранит информацию о дате истечения, дате создания и статусе отзыва (Revoked).

---

## Схема связей

1. **Chat** 1 --- N **Message** (Один чат содержит много сообщений).
2. **Chat** 1 --- N **ChatParticipant** (В чате много участников).
3. **AppUser**(Infrastructure) 1 --- N **ChatParticipant** (Пользователь участвует в разных чатах).
4. **Message** 0 --- 1 **Message** (У сообщения может быть ссылка на другое сообщение — ответ).
