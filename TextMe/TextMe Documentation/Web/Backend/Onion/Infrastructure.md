# Документация Инфраструктурного Слоя в TextMe (Infrastructure)

Инфраструктурный слой отвечает за реализацию всех технических аспектов приложения, включая хранение данных, безопасность и интеграцию с внешними сервисами.

---

## 1. Репозитории (Repositories)
Репозитории инкапсулируют логику доступа к данным, изолируя приложение от деталей работы с **EF Core**.

- **ChatRepository**: Управляет созданием чатов, поиском участников и получением списка бесед. Включает сложную логику фильтрации для получения последних сообщений.
- **MessageRepository**: Работает с сообщениями, включая их редактирование, удаление и поиск по истории чата.
- **RefreshTokenRepository**: Отвечает за хранение и валидацию токенов обновления, обеспечивая безопасность сессий.
- **AccountRestoreRepository**: Логика восстановления доступа к аккаунтам.

---

## 2. Безопасность и JWT
Система аутентификации построена на стандарте **JWT (JSON Web Token)**.

- **JwtTokenService**: Генерирует токен доступа (AccessToken) и токен обновления (RefreshToken). 
- **EncryptionService**: Реализует симметричное шифрование (AES) для защиты содержимого сообщений перед их сохранением в базу данных.

---

 ## 3. Real-time уведомления и Хабы
Для обеспечения «живого» общения используется **SignalR**.

- **ChatHub**: Главная точка входа для WebSocket-соединений. Обрабатывает подключение пользователей, их статус («В сети») и передачу событий печатания (typing).
 - **HubMessageRealtimeNotifier**: Инфраструктурная реализация интерфейса `IMessageRealtimeNotifier`. Она отвечает за рассылку уведомлений о новых сообщениях, входящих звонках и других событиях конкретным пользователям.

 ---

## 4. Сервисы (Services)
Специализированная логика, требующая внешних интеграций или сложной обработки.

- **CloudinaryStorage**: Сервис для загрузки и хранения медиафайлов (изображения, аудио) в облачном хранилище Cloudinary.
- **UserPresenceManager / Service**: Отслеживает статус пользователей (Online/Offline) в реальном времени, используя Redis или in-memory хранилище.
- **RssNewsFeedService**: Сервис для интеграции внешних новостных лент прямо в интерфейс мессенджера.

---

## 5. Работа с данными (Data & Stores)
- **AppDbContext**: Основной контекст базы данных, где настроены связи между таблицами (Fluent API).
 - **UserStore**: Кастомная реализация `IUserStore` для ASP.NET Identity, позволяющая полностью контролировать процесс хранения и извлечения данных о пользователях (`AppUser`).

 ---

##  Техническая реализация сущностей

### AppUser (Infrastructure Entity)
AppUser (Пользователь)
 Наследуется от `IdentityUser`. Содержит расширенную информацию о профиле.

 - **AvatarUrl**: Ссылка на фото профиля пользователя.
- **FirstName / LastName**: Имя и фамилия.
- **LastSeenAt**: Время последнего онлайна.
- **ShareOnlineStatus**: Настройка приватности отображения статуса "В сети".
- **CreatedAt / UpdatedAt**: Таймстампы жизненного цикла аккаунта.

 ---

### Хелперы (Helpers)
 В слое инфраструктуры содержатся специализированные технические утилиты:
 -   **RssParsingHelper**: Сложный инструмент для парсинга RSS-лент. Он включает в себя логику очистки HTML от тегов (PlainText), извлечение изображений из различных форматов метаданных (OpenGraph, Media RSS) и нормализацию ссылок. Это позволяет корректно отображать новости из внешних источников внутри приложения.
