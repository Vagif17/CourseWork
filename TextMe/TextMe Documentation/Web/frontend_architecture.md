# Архитектура Frontend приложения (Web)

Приложение построено с использованием методологии **Feature-Sliced Design (FSD)**. Это архитектурный паттерн, который разбивает проект на слои, слайсы и сегменты для повышения масштабируемости и чистоты кода.

## Структура слоев

Проект разделен на следующие уровни (от нижнего к верхнему):

1.  **Shared**: Переиспользуемые модули, не знающие о бизнес-логике.
    *   `ui/`: Базовые компоненты (кнопки, инпуты, модалки, спиннеры).
    *   `api/`: Сервисы для работы с API и SignalR хабы.
    *   `lib/`: Утилиты, кастомные хуки, валидаторы.
    *   `config/`: Константы и глобальные конфигурации.
    *   `assets/`: Изображения, иконки и статические ресурсы.

2.  **Entities**: Бизнес-сущности (User, Chat, Message, News).
    *   Содержат логику, специфичную для конкретной модели данных.
    *   `ui/`: Компоненты отображения данных (`ChatItem`, `MessageItem`, `NewsChatCard`, `ProfileAvatarBlock`).

3.  **Features**: Действия пользователя, приносящие бизнес-ценность.
    *   `auth/`: Процессы входа, регистрации и восстановления пароля.
    *   `chat/`: Логика сообщений (`MessageInput`), создание чатов и групп, шаринг контента.
    *   `profile/`: Управление данными аккаунта и смена пароля.

4.  **Widgets**: Сложные блоки интерфейса, собираемые из сущностей и фич.
    *   `chat-section`: Основной виджет чата и список чатов.
    *   `SidebarNav`: Боковая навигационная панель.
    *   `NewsFeedSection`: Лента новостей.
    *   `ProfileSection`: Секция управления профилем.
    *   `SettingsSection`: Настройки приложения и темы.

5.  **Pages**: Целые страницы приложения.
    *   Компонуют виджеты в готовые представления.
    *   Примеры: `AuthPage`, `HomePage`.

6.  **App**: Настройки всего приложения.
    *   Провайдеры (Redux, Theme, Context).
    *   Глобальные стили.
    *   Инициализация приложения (`main.tsx`).

## Основные принципы

- **Направленность зависимостей**: Модули на верхних слоях могут импортировать модули только с нижних слоев. Например, `Widget` может использовать `Feature`, но не наоборот.
- **Публичное API**: Каждый слайс имеет файл `index.ts`, который является единственной точкой входа. Это защищает от утечки внутренних деталей реализации.
- **Изоляция стилей**: Стили выносятся в отдельные файлы (например, `.module.css` или `.styles.ts`), чтобы логика компонента оставалась лаконичной.
