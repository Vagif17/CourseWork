# Архитектура Мобильного приложения (React Native)

Мобильное приложение TextMe организовано по принципам **Feature-Sliced Design (FSD)**, что позволяет избежать громоздких файлов и четко разделить ответственность между компонентами.

## Слои архитектуры

1.  **Shared**: Фундамент приложения.
    *   `src/shared/ui/`: Базовые компоненты интерфейса.
    *   `src/shared/api/`: Сервисы запросов и SignalR хабы.
    *   `src/shared/lib/`: Общие хуки, утилиты и вспомогательные функции.
    *   `src/shared/config/`: Константы, темы и глобальный контекст.

2.  **Entities**: Представление бизнес-данных.
    *   `src/entities/message/`: Компонент `MessageItem`, `NewsChatCard` и логика отображения различных типов сообщений.

3.  **Features**: Интерактивные возможности.
    *   `src/features/chat/`: Компонент `ChatInput` — сложная логика ввода сообщений, вложений и записи аудио.
    *   Другие фичи (поиск, шаринг) реализованы в виде модальных окон в `shared/ui`.

4.  **Widgets**: Крупные функциональные блоки.
    *   `chat-list`: Список чатов с SignalR-логикой.
    *   `chat-window`: Интерфейс чата с сообщениями и вводом.
    *   `news-feed`: Лента новостей.
    *   `profile-section`: Управление профилем и настройками.
    *   `auth-form`: Формы входа, регистрации и восстановления.
    *   Все виджеты используют `styled-components` в файлах `.styles.ts`.

5.  **Pages (Routing)**: Экраны приложения.
    *   Концептуально страницы представлены виджетами, а маршрутизация сосредоточена в `src/app/` (используется Expo Router).
    *   Маршруты представляют собой тонкие обертки: `src/app/(tabs)/`, `src/app/(auth)/`, `src/app/chat/[id]/index.tsx`.

6.  **App Layer**: Глобальная конфигурация.
    *   `src/app/providers/`: Централизованные провайдеры (Theme, Auth).
    *   `src/shared/config/constants/CallContext.tsx`: Провайдер звонков (WebRTC).
    *   `src/app/_layout.tsx`: Корневой макет и навигационный стек.

## Особенности реализации

-   **Разделение стилей**: Вся визуальная составляющая вынесена из файлов разметки в отдельные файлы `.styles.ts` с использованием `styled-components`. Это делает код компонентов чистым и фокусирует его на логике.
-   **Компактность кода**: Благодаря декомпозиции на виджеты и фичи, файлы не превышают разумных пределов (избегаем "God Objects" по 500+ строк).
-   **Типизация**: Строгое использование TypeScript для всех пропсов и моделей данных.
-   **Public API**: Каждый модуль предоставляет только необходимые данные через `index.ts`, скрывая внутреннюю структуру папок.
